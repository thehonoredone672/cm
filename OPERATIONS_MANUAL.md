# CodeMatch Operations Manual

This guide is intended for system operators, administrators, and DevOps engineers managing the CodeMatch platform.

---

## 1. Daily Operations

### Health Verification Checklists
To verify overall system health:
1. Ensure the PostgreSQL connection is stable:
   ```bash
   npx prisma db pull
   ```
2. Verify Judge0 sandbox compiles correctly:
   ```bash
   curl -I https://api.judge0.com/statuses
   ```

### Logs & Telemetry Monitoring
- Express logs are outputted to the standard shell console.
- Check PM2 / Docker logs:
  ```bash
  pm2 logs codematch-backend
  ```

---

## 2. User & Moderation Management

### Administrator Roles Assignment
- Users can be updated using Prisma CLI or SQL Admin panel:
  ```sql
  UPDATE "User" SET role = 'ADMIN' WHERE email = 'operator@codematch.com';
  ```

---

## 3. Database Maintenance & Operations

### Manual Backups (pg_dump)
To backup the current PostgreSQL schema:
```bash
pg_dump -U postgres -d codematch -F c -b -v -f "/var/backups/codematch_backup.sql"
```

### Manual Database Restores
```bash
pg_restore -U postgres -d codematch -v "/var/backups/codematch_backup.sql"
```

### Prisma Schema Migrations
Always commit migration directories generated by Prisma:
```bash
npx prisma migrate dev --name your_migration_name
```
To run pending migrations on production:
```bash
npx prisma migrate deploy
```
